LoopID Concept

LoopID is a unique identifier system used to track items in loops or iterations, particularly in React applications. It ensures each item has a consistent identifier across re-renders, improving performance and debugging capabilities.

Key Features

  • Uniqueness: Each loop iteration gets a unique identifier
  • Consistency: Same data produces same ID across renders
  • Traceability: Easy debugging and tracking of list items
  • Performance: Helps React optimize re-rendering of list items

Implementation Example

// LoopID Generation Pattern
const generateLoopId = (prefix, index, timestamp = Date.now()) => {
  return `${prefix}_${index}_${timestamp}`;
};

// Usage in React Components
const ItemList = ({ items }) => {
  return (
    <div>
      {items.map((item, index) => {
        const loopId = generateLoopId('item', index, item.createdAt);
        return (
          <div key={loopId} data-loop-id={loopId}>
            <h3>{item.title}</h3>
            <p>{item.description}</p>
          </div>
        );
      })}
    </div>
  );
};

Use Cases in L4L Project

  • Issue Tracking: Each issue in the list gets a unique loop ID
  • Product Catalogs: Products in collections use loop IDs for consistent rendering
  • User Management: User list items maintain their identity across updates

LinkUtil Utility

LinkUtil is a comprehensive utility module that handles various link-related operations in web applications, including URL generation, validation, navigation management, and file download operations.

Core Functions

const LinkUtil = {
  // Generate dynamic URLs with parameters
  generateUrl: (baseUrl, params = {}) => {
    const url = new URL(baseUrl);
    Object.keys(params).forEach(key => {
      if (params[key] !== null && params[key] !== undefined) {
        url.searchParams.set(key, params[key]);
      }
    });
    return url.toString();
  },

  // Validate URL format
  isValidUrl: (string) => {
    try {
      new URL(string);
      return true;
    } catch (_) {
      return false;
    }
  },

  // Create download links for file exports
  createDownloadLink: (data, filename, type = 'text/csv') => {
    const blob = new Blob([data], { type: `${type};charset=utf-8;` });
    const link = document.createElement('a');
    link.href = URL.createObjectURL(blob);
    link.download = filename;
    return link;
  }
};

Use Cases in L4L Project

  • File Downloads: Handles CSV export functionality
  • Navigation Management: Manages internal and external links
  • URL Parameter Handling: Processes query parameters for filters

useDeepLink Hook

useDeepLink is a custom React hook that manages deep linking functionality, allowing components to handle URL changes, navigation state, route parameters, and state persistence across page refreshes.

Key Features

  • URL State Management: Syncs component state with URL parameters
  • Navigation Control: Programmatic navigation with history management
  • State Persistence: Maintains state across page refreshes
  • Breadcrumb Support: Tracks navigation history

Basic Implementation

const useDeepLink = (initialPath = '/') => {
  const router = useRouter();
  const [currentPath, setCurrentPath] = useState(initialPath);
  const [params, setParams] = useState({});
  const [isNavigating, setIsNavigating] = useState(false);

  // Navigate to a specific path with parameters
  const navigateTo = useCallback((path, options = {}) => {
    const { params: newParams, replace = false } = options;
    
    let fullPath = path;
    if (newParams && Object.keys(newParams).length > 0) {
      const searchParams = new URLSearchParams(newParams);
      fullPath = `${path}?${searchParams.toString()}`;
    }

    if (replace) {
      router.replace(fullPath);
    } else {
      router.push(fullPath);
    }
  }, [router]);

  return {
    currentPath,
    params,
    isNavigating,
    navigateTo,
    goBack: () => router.back(),
    isActive: (path) => currentPath === path
  };
};

Use Cases in L4L Project

  • Issue Detail Navigation: Direct links to specific issues
  • Filter State Management: Maintains filter state in URL
  • Modal Deep Linking: Links directly to modals with specific data
  • Breadcrumb Navigation: Tracks user navigation path

CreateLoopModal Component

CreateLoopModal is a reusable modal component designed specifically for creating items within loops or collections. It provides a standardized interface for item creation with validation, state management, and user feedback.

Key Features

  • Dynamic Fields: Conditional field display based on form state
  • Validation System: Real-time field validation with custom rules
  • Loading States: Handles submission and loading states
  • Loop Context: Maintains context about the parent collection

Component Structure

const CreateLoopModal = ({
  isOpen,
  onClose,
  onSubmit,
  title = "Create New Item",
  fields = [],
  validationRules = {},
  loopContext = null
}) => {
  const [formData, setFormData] = useState({});
  const [errors, setErrors] = useState({});

  // Handle form submission
  const handleSubmit = async (e) => {
    e.preventDefault();
    
    if (!validateForm()) return;

    try {
      const result = await onSubmit({
        ...formData,
        loopContext,
        createdAt: new Date().toISOString(),
        id: Date.now()
      });

      if (result.success) {
        onClose();
        setFormData({});
      }
    } catch (error) {
      setErrors({ submit: 'Failed to create item.' });
    }
  };

  return (
    <Modal isOpen={isOpen} onClose={onClose}>
      <form onSubmit={handleSubmit}>
        {/* Dynamic field rendering */}
        {fields.map((field) => (
          <FormField
            key={field.name}
            {...field}
            value={formData[field.name] || ''}
            onChange={(value) => handleFieldChange(field.name, value)}
            error={errors[field.name]}
          />
        ))}
      </form>
    </Modal>
  );
};

// Complete Integration: Dynamic Issue Management System
const IssueManagementSystem = () => {
  const { params, updateParams, navigateTo } = useDeepLink({
    persistState: true,
    defaultParams: { view: 'list', filter: 'all' }
  });

  const [issues, setIssues] = useState([]);
  const [showCreateModal, setShowCreateModal] = useState(false);

  // Handle issue creation with loop tracking
  const handleCreateIssue = async (issueData) => {
    const loopId = `issue_${Date.now()}`;
    const issueWithLoopId = {
      ...issueData,
      loopId,
      createdAt: new Date().toISOString()
    };

    const result = await createIssue(issueWithLoopId);
    if (result.success) {
      setIssues(prev => [...prev, result.data]);
      
      // Navigate to the newly created issue using LinkUtil
      const detailUrl = LinkUtil.generateUrl('/issues/detail', {
        id: result.data.id,
        loopId
      });
      
      navigateTo(detailUrl);
    }

    return result;
  };

  return (
    <div>
      {/* Issue List with Loop IDs */}
      {issues.map((issue, index) => {
        const loopId = issue.loopId || `fallback_${issue.id}_${index}`;
        return (
          <IssueCard
            key={loopId}
            issue={issue}
            loopId={loopId}
            onClick={() => navigateTo(`/issues/${issue.id}`, {
              params: { loopId }
            })}
          />
        );
      })}

      {/* CreateLoopModal Integration */}
      <CreateLoopModal
        isOpen={showCreateModal}
        onClose={() => setShowCreateModal(false)}
        onSubmit={handleCreateIssue}
        title="Create New Issue"
        fields={issueFields}
        loopContext={{
          parentCollection: 'issues',
          currentCount: issues.length,
          filters: params
        }}
      />
    </div>
  );
};
